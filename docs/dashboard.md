[기획안] 개인용 와인 셀러 재고 관리 웹 앱 (MVP)
1. 프로젝트 개요
목적: 구글 시트 기반의 와인 재고 데이터를 실시간으로 확인하고, 음용 시 버튼 클릭 한 번으로 상태를 변경하는 최적화된 모바일 웹 구축.

배경: 기존 노션(Notion) 관리 방식의 속도 저하 및 복잡한 UI를 개선하여, '재고 확인'과 '소비 기록'에만 집중된 고속 인터페이스가 필요함.

2. 주요 기능 (Core Features)
실시간 재고 리스트: 구글 시트에서 Status가 In Stock인 데이터만 필터링하여 노출.

원터치 상태 변경: 리스트의 각 와인 항목 옆에 [Drink! 🍷] 버튼 배치. 클릭 시 시트의 Status가 Consumed로 즉시 업데이트됨.

낙관적 UI (Optimistic UI) 적용: 버튼 클릭 시 서버 응답을 기다리지 않고 리스트에서 즉시 제거하여 사용자 체감 속도 극대화.

핵심 정보 요약: 이름, 빈티지, 생산자 등 최소한의 필수 정보만 카드 형태로 노출하여 가독성 확보.

3. 화면 설계 (UI/UX)
레이아웃: 모바일 우선(Mobile-First) 디자인, 수직 스크롤 리스트 형태.

와인 카드 구성:

상단: 와인 이름 (Bold)

중간: 빈티지 | 생산자 | 생산지

하단: [Drink / 마셨음] 버튼 (눈에 잘 띄는 색상 및 큰 클릭 영역)

검색창 (Optional): 리스트 상단에 와인 이름으로 검색할 수 있는 퀵 서치 바.

4. 데이터 요구사항 (Data Integration)
데이터 소스: 구글 스프레드시트 (Wine Cellar 파일)

필요 필드(열):

Name (와인 이름)

Vintage (생산 연도)

Producer (생산자)

Status (상태: In Stock / Consumed)

RowID (업데이트를 위한 행 번호)

5. 기술 스택 및 배포 계획
Frontend/Backend: Next.js (App Router)

API: Google Sheets API (v4)

Deployment: Vercel (CI/CD 자동화)

6. 기획자 전달용 핵심 메시지
"이 페이지의 핵심은 **'속도'**와 **'단순함'**입니다. 복잡한 입력창 없이, 현재 셀러에 있는 와인을 쭉 보고 마시는 순간 버튼 하나로 리스트에서 지워버리는(Consumed 처리) 동작이 가장 매끄럽게 구현되어야 합니다. 디자인은 깔끔한 와인 라벨 스타일의 대시보드를 지향합니다."
